Selection bug in search

Reported behavior

When you try to select one result, it selects all of them immediately.

That means:
	•	the selection state is not isolated per item
	•	one tap is behaving like a bulk-select
	•	selection is incorrectly triggering a send state

Intended behavior
	•	tapping one result should select only that one result
	•	it should not auto-send anything
	•	it should let the user review or compose the intended outgoing request/message flow before sending

Send/request flow bug

Reported behavior

After selection, the interface effectively says something like “sent”, but:
	•	no visible message is prepared
	•	no outgoing message thread appears
	•	no connection request is received on the other end
	•	no inbound message asking to connect is received
	•	nothing actually arrives to the recipient

You specifically noted:
	•	it does not send a message
	•	it just kind of says sent
	•	you don’t get to see the message
	•	the message that was already designed is not being presented and then sent
	•	no message asking to be connected is received on the other side

What that means functionally

The UI is likely showing a success state without a real completed send pipeline.

In practical terms, one or more of these is failing:
	•	no request record is being created
	•	no outgoing message object is being created
	•	no recipient-facing inbox/system message is being created
	•	no notification is being triggered
	•	the compose/send flow is being bypassed entirely
